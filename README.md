# 网站可用性实时检测工具

本仓库为一个纯前端的「网站可用性实时检测」工具（单文件实现），用于在浏览器环境下对多个站点做一次性可用性检测并记录日志。适合本地使用、学习和演示用途；不含后端组件。

演示地址：https://urltest.937788.xyz/

主要功能
- 从浏览器导出的书签 HTML 导入网址并自动加入检测列表。
- 单次并发检测（支持并发限制、停止操作、HEAD→GET回退）。
- 日志实时记录与导出为 CSV（含中文表头与中文文件名前缀）。

文件说明
- `index.html`：完整的前端源码（UI + JS 逻辑 + 样式）。
- `README.md`：本说明文件。

使用方法
1. 在本地打开 `index.html`（双击或通过本地静态服务器）。
2. 在左侧输入框添加单个网址，或使用“导入收藏夹”选择浏览器导出的 bookmarks HTML 文件批量导入。
3. 点击“监测一次”开始检测，检测过程中可点击“停止检测”中止当前未完成的请求。
4. 在右侧可查看日志并通过“导出日志”保存为 CSV 文件。

注意事项
- 本工具在浏览器环境下运行，请注意跨域（CORS）限制可能导致无法获取远端 HTTP 状态码或出现 `Failed to fetch`。
- 为避免页面卡顿，工具使用了并发限制（默认 5）。在大批量 URL 场景建议适当降低 `CONCURRENCY_LIMIT`。
- 若希望长期稳定记录大量日志，建议配合后端或数据库存储替代浏览器内存。
